rok - Run One, Know All

Execute multi-step tasks from JSON - the ultimate automation tool for developers and AI agents.

Get Started GitHub

Installation

From crates.io

cargo install rok-cli

From Source

git clone https://github.com/ateeq1999/rok
cd rok
cargo install --path .

Verify Installation

rok --version

Quick Start

Basic Usage

# Run from file
rok -f task.json

# Run from stdin
echo '{"steps":[{"type":"bash","cmd":"echo hello"}]}' | rok

# Run from inline JSON
rok -j '{"steps":[{"type":"bash","cmd":"echo hello"}]}'

Example: File Operations

{
  "steps": [
    { "type": "mkdir", "path": "src/components" },
    { "type": "write", "path": "src/components/Hello.tsx", "content": "export const Hello = () => 
Hello World
" },
    { "type": "bash", "cmd": "npm run build" }
  ]
}

Steps

rok provides 20+ built-in step types organized by category:

File Operations

read - Read file contents
write - Write content to file
mv - Move/rename files
cp - Copy files
rm - Remove files
mkdir - Create directories
patch - Apply edits to files

Search & Analysis

grep - Search in files
scan - Scan directory tree
extract - Extract data from files
summarize - Summarize code
lint - Lint code
diff - Show differences

Other

bash - Run shell commands
template - Render templates
git - Git operations
http - HTTP requests
snapshot - Save snapshot
restore - Restore snapshot

References

Pass data between steps using refs:

{
  "steps": [
    { "type": "scan", "path": "./src", "depth": 2 },
    { "type": "each", "over": {"ref": 0, "pick": "tree.*"}, "step": { "type": "bash", "cmd": "echo {{item}}" }}
  ]
}

Conditionals

{
  "steps": [
    { "type": "grep", "pattern": "TODO", "path": "./src" },
    {
      "type": "if",
      "condition": { "type": "grep_has_results", "ref": 0 },
      "then": [
        { "type": "bash", "cmd": "echo Found TODOs!" }
      ]
    }
  ]
}

Templates

Create reusable templates with props:

# Create a new template
rok init-template my-component

# List available templates
rok templates

Templates support:

  • Custom props with validation
  • Inheritance (extends field)
  • Derived transforms (pluralize, camelcase, etc.)

Task Files

Save and reuse tasks:

# Save current payload as a named task
rok -f task.json save my-task -d "Build the project"

# Run saved task
rok run my-task

# List all tasks
rok list

Environment Variables

Use {{env.VAR_NAME}} in any string field:

{
  "steps": [
    { "type": "bash", "cmd": "echo {{env.USER}}" }
  ]
}

Timeouts & Retries

{
  "steps": [
    {
      "type": "bash",
      "cmd": "make build",
      "timeout_ms": 60000,
      "retry": {
        "count": 3,
        "delay_ms": 2000,
        "backoff": true
      }
    }
  ]
}

CLI Commands

CommandDescription
rok templatesList available templates
rok init-template [name]Create a new template
rok validate-template [path]Validate a template schema
rok run <task>Run a saved task
rok save <name>Save current payload as task
rok listList saved tasks
rok edit <task>Edit a saved task
rok watchWatch files and re-run
rok historyShow execution history
rok replay [id]Replay a previous run

Options

OptionDescription
-f, --file FILEPath to JSON file
-j, --json JSONInline JSON payload
-o, --output FORMATOutput format: json, pretty, silent
-v, --verboseEnable verbose output
-q, --quietSuppress output
--dry-runPreview without executing

JSON Schema

Complete task file schema:

{
  "name": "my-task",
  "description": "Task description",
  "version": "1.0.0",
  "options": {
    "cwd": ".",
    "stop_on_error": true,
    "timeout_ms": 30000,
    "env": { "KEY": "value" }
  },
  "props": {
    "version": "1.0.0"
  },
  "steps": []
}
