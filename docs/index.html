rok

Run One, Know All - Execute multi-step tasks from JSON

Get Started GitHub

What is rok?

rok is a powerful CLI tool that executes multi-step tasks defined in JSON. It's designed for developers and AI agents who need to automate complex workflows.

🚀 20+ Step Types

File operations, search, code analysis, HTTP requests, git operations, and more.

🔗 Data Flow

Pass data between steps using references and conditionals.

📋 Templates

Create reusable templates with props and inheritance.

⏱️ Timeouts & Retries

Built-in support for per-step timeouts and retry logic.

🌐 Environment Variables

Use {{env.VAR}} syntax in any field.

📚 Self-Documenting

Built-in docs server with rok serve.

Installation

From crates.io

cargo install rok-cli

From Source

git clone https://github.com/ateeq1999/rok
cd rok
cargo install --path .

Verify

rok --version

Quick Start

Run from File

rok -f task.json

Run from stdin

echo '{"steps":[{"type":"bash","cmd":"echo hello"}]}' | rok

Run from Inline JSON

rok -j '{"steps":[{"type":"bash","cmd":"echo hello"}]}'
Example: Create a React Component
{
  "steps": [
    { "type": "mkdir", "path": "src/components" },
    { "type": "template", "builtin": "react-component", "output": "src/components/Button.tsx", "vars": { "name": "Button" } }
  ]
}

Task File Structure

A complete rok task file supports the following fields:

{
  "name": "my-task",
  "description": "Task description",
  "version": "1.0.0",
  "author": "Your Name",
  
  "options": {
    "cwd": ".",
    "stop_on_error": true,
    "timeout_ms": 30000,
    "env": { "KEY": "value" }
  },
  
  "props": {
    "version": "1.0.0"
  },
  
  "steps": []
}

Top-Level Fields

FieldTypeDescription
namestringTask name
descriptionstringTask description
versionstringTask version
authorstringTask author
optionsobjectExecution options
propsobjectReusable variables
stepsarrayArray of steps to execute

Options

OptionDefaultDescription
cwd"."Working directory
stop_on_errortrueStop on first error
timeout_ms30000Global timeout
env{}Environment variables

Steps Overview

Steps are the core building blocks of rok. Each step performs a specific action.

All Step Types

CategorySteps
Fileread, write, mv, cp, rm, mkdir, patch
Searchgrep, scan, extract
Analysissummarize, lint, diff
Systembash, http, git
Controlif, each, parallel
Templatetemplate
Snapshotsnapshot, restore

All steps support these optional fields:

  • id - A unique identifier for referencing

bash

Execute shell commands.

{
  "type": "bash",
  "cmd": "npm run build",
  "timeout_ms": 60000,
  "retry": {
    "count": 3,
    "delay_ms": 1000,
    "backoff": true
  }
}

Fields

FieldTypeRequiredDescription
cmdstringYesShell command to execute
idstringNoStep identifier
timeout_msnumberNoTimeout in milliseconds
retryobjectNoRetry configuration

Retry Configuration

FieldTypeDefaultDescription
countnumber3Number of retry attempts
delay_msnumber1000Delay between retries
backoffbooleanfalseExponential backoff

read

Read file contents.

{
  "type": "read",
  "path": "src/main.rs",
  "max_bytes": 1048576,
  "encoding": "utf-8"
}
FieldTypeRequiredDescription
pathstringYesFile path to read
max_bytesnumberNoMaximum bytes to read
encodingstringNoFile encoding (default: utf-8)

write

Write content to a file.

{
  "type": "write",
  "path": "output.txt",
  "content": "Hello World",
  "create_dirs": true
}
FieldTypeRequiredDescription
pathstringYesOutput file path
contentstringYesContent to write
create_dirsbooleanNoCreate parent directories (default: true)

File Operations: mv, cp, rm, mkdir

mv - Move/Rename

{
  "type": "mv",
  "from": "old.txt",
  "to": "new.txt"
}

cp - Copy

{
  "type": "cp",
  "from": "source.txt",
  "to": "dest.txt",
  "recursive": true
}

rm - Remove

{
  "type": "rm",
  "path": "temp/",
  "recursive": true
}

mkdir - Create Directory

{
  "type": "mkdir",
  "path": "src/components"
}

patch

Apply edits to a file.

{
  "type": "patch",
  "path": "src/main.rs",
  "edits": [
    { "find": "fn old()", "replace": "fn new()" },
    { "find": "// TODO: implement", "replace": "// Done" }
  ]
}
FieldTypeRequiredDescription
pathstringYesFile to patch
editsarrayYesArray of edit objects

grep

Search for patterns in files.

{
  "type": "grep",
  "pattern": "TODO",
  "path": "./src",
  "ext": ["rs", "js", "ts"],
  "regex": true,
  "context_lines": 2
}
FieldTypeRequiredDescription
patternstringYesSearch pattern
pathstringYesDirectory to search
extarrayNoFile extensions to filter
regexbooleanNoUse regex (default: false)
context_linesnumberNoLines before/after match

scan

Scan directory tree and get file listing.

{
  "type": "scan",
  "path": "./src",
  "depth": 3,
  "include": ["*.rs", "*.toml"],
  "output": "tree"
}
FieldTypeRequiredDescription
pathstringYesDirectory to scan
depthnumberNoMax depth (default: 3)
includearrayNoFile patterns to include
outputstringNoOutput format: summary, tree, list

extract

Extract data from JSON/YAML files using JSONPath.

{
  "type": "extract",
  "path": "package.json",
  "pick": ["name", "version", "dependencies"]
}
FieldTypeRequiredDescription
pathstringYesFile to extract from
pickarrayNoKeys to extract

summarize

Summarize code structure (exports, functions, imports).

{
  "type": "summarize",
  "path": "./src",
  "focus": "exports"

Custom Templates

Create your own templates in .rok/templates/

Create a Template

rok init-template my-template

Template Schema (.rok-template.json)

{
  "name": "my-template",
  "description": "My custom template",
  "version": "1.0.0",
  "author": "Your Name",
  "tags": ["custom", "template"],
  "extends": "base-template",
  "output": [
    { "from": "template.txt", "to": "{{name}}.txt" }
  ],
  "props": {
    "name": {
      "prop_type": "string",
      "required": true,
      "description": "The name"
    }
  }
}

  "args": ["-m", "commit message"]

http

Make HTTP requests.

{
  "type": "http",
  "method": "GET",
  "url": "https://api.github.com/users/{{env.USER}}",
  "headers": {
    "Authorization": "Bearer {{env.TOKEN}}",
    "Accept": "application/json"
  },
  "expect_status": 200
}
FieldTypeRequiredDescription
methodstringYesHTTP method (GET, POST, etc.)
urlstringYesRequest URL

bodystringNoRequest body
expect_statusnumberNoExpected status code

snapshot & restore

Save and restore file snapshots for safety during risky operations.

{
  "steps": [
    { "type": "snapshot", "path": ".", "snapshot_id": "before-refactor" },
    { "type": "bash", "cmd": "rm -rf src" },
    {
      "type": "if",
      "condition": { "type": "step_failed", "ref": 1 },
      "then": [
        { "type": "restore", "snapshot_id": "before-refactor" }
      ]
    }
  ]
}

References

Pass data between steps using the ref system.

Basic Reference

{
  "steps": [
    { "type": "scan", "path": "./src", "depth": 2 },
    { "type": "each", "over": {"ref": 0, "pick": "tree.*"}, "step": { "type": "bash", "cmd": "echo {{item}}" } }
  ]
}

JSONPath

{
  "steps": [
    { "type": "extract", "path": "package.json", "pick": ["name", "version"] },
    { "type": "bash", "cmd": "echo {{ref:0.name}}" }
  ]
}

Conditionals (if)

Execute steps conditionally based on results.

{
  "type": "if",
  "condition": { "type": "grep_has_results", "ref": 0 },
  "then": [
    { "type": "bash", "cmd": "echo Found TODOs!" }
  ],
  "else": [
    { "type": "bash", "cmd": "echo No TODOs found" }
  ]
}

Condition Types

TypeDescription
existsCheck if path exists
containsCheck if file contains pattern
grep_has_resultsCheck if grep found matches
step_okCheck if step succeeded
step_failedCheck if step failed
file_changedCheck if file changed

Loops (each) & Parallel

each - Iterate

{
  "type": "each",
  "over": ["a", "b", "c"],
  "as": "item",
  "parallel": false,
  "step": {
    "type": "bash",
    "cmd": "echo {{item}}"
  }
}

parallel - Run Simultaneously

{
  "type": "parallel",
  "steps": [
    { "type": "bash", "cmd": "task1" },
    { "type": "bash", "cmd": "task2" },
    { "type": "bash", "cmd": "task3" }
  ]
}

Environment Variables

Use {{env.VAR_NAME}} in any string field.

{
  "steps": [
    { "type": "bash", "cmd": "echo Hello {{env.USER}}" },
    { "type": "http", "url": "https://api.{{env.DOMAIN}}/data" }
  ]
}

Timeouts & Retries

Add reliability to commands with timeouts and retries.

{
  "type": "bash",
  "cmd": "make build",
  "timeout_ms": 60000,
  "retry": {
    "count": 3,
    "delay_ms": 2000,
    "backoff": true
  }
}

Task Commands

Save and manage reusable tasks.

Save a Task

rok -f task.json save build -d "Build the project"

Run a Task

rok run build

List Tasks

rok list

Edit a Task

rok edit build
